# Authoring CLAUDE.md

- CLAUDE.md carries only the facts every session needs up front: what the project is, stack, commands, environment, repo map.
- Facts, not behavior. Anything telling the agent *how to act* in an area belongs in a scoped `.claude/rules/` file, never here.
- If a fact is already owned by a rule or a doc, link it or omit it — don't restate it.
