Rules:
- The agent appends here ONLY when a discovery is genuinely actionable and would otherwise be lost — and it says so in-session when it does. No silent accumulation.
- **Never read as a task list.** Nothing here is planned work; the `/feature` flow never pulls slices from it.
- The user prunes it: promote a real item into `docs/triage.md` (or act on it), delete the rest. An unpruned queue is just noise with a new name.
